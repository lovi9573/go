Hello World, part II (using variables)

The following example shows how to create a string variable containing hello world, and then printing that to the console. There are two keywords used to create variables: var and cons. You use cons for constants, and var for everything else. Thus, you can create variables as follows:

  • [cons or var] [Name] [Type]
  • [cons or var] [Name] [Type] = [what you want to assign goes here]
Or, if you're creating a variable inside a function you can use this format:
  • [Name] := [what you want to assign goes here]
Here's a simple example:

package main

import "fmt"

func main() {
    var blank string
    var message string = "Hello World"
    fast := "This format is simple..."
    
    fmt.Println(blank)
    fmt.Println(message)
    fmt.Println(fast)
}

The output: 


Hello World
This format is simple...

Creating Integer Variables

Here's another example, but creating integer variables. Notice that the compiler is smart enough to figure out type depending on what you assign to a specific variable. Also, if you're coming from Java, notice the comma in the Println instead of a plus symbol: 

package main

import "fmt"

func main() {
 //  int types: int8  int16  int32  int64 or just int
//unsiged ints: uint8 uint16 uint32 uint64 or just uint
var num1 int = 4 
var num2 int = 6 
var num3 = 3 
var num4 = 2 
num5 := 9

var sum int=int(num1+num2+num3+num4+num5)
fmt.Println("Sum: ", sum)
}

The output: 

Sum:  24

More Concise

Finally, to take that one step further, this would be even more concise. Why type it, if the compiler can figure it out:

package main

import "fmt"

func main() {
    num1, num2, num3, num4, num5 := 4,6,3,2,9
    sum := num1+num2+num3+num4+num5
    fmt.Println("Sum: ", sum)
}

The output: 

Sum:  24

Let the Compiler figure it out

This is where is starts getting really different than Java, and really cool in my opinion (true and false are boolean values here, not strings):

package main

import "fmt"

func main() {
    var a,b,c,d,e = 1, 3.14, true, false, "Wow, this is a string"
    fmt.Println(a,b,c,d,e)
}

The output: 

1 3.14 true false Wow, this is a string

Global Variables and More

The following example shows how to declare different types of variables outside the methods (globally) and how you could "declare" a variable inside the method to have the same name as a global variable without changing the global variable:

package main

import "fmt"

var i int
var x = 9   
var y int = 12

var z, w int    //cant be var z int, w int

var toe int = 0
var foo, bar int = 1, 2
var apple, green = 3, "Four"

var(
    Im bool = false
    Cookie = "Oatmeal"
)

func main(){
    fmt.Println("\ny =", y) //should print 12
    y = 13
    fmt.Println("y =", y) //should print 13

    y := 14               //creates a local y thats "untyped" 
    fmt.Println("y =", y) //should print 14
    tmp()                 //should print 13

    i = 5
    z, w = 7, 8

    k := "Not typed" // cant be k = "Not type" 

    
    fmt.Println("toe  =", toe)
    fmt.Println("foo  =", foo)
    fmt.Println("bar  =", bar)
    fmt.Println("apple  =", apple)
    fmt.Println("i =", i)
    fmt.Println("green  =", green)
    fmt.Println("Im  =", Im)
    fmt.Println("Cookie  =", Cookie) 
    fmt.Println("k =", k, "\n")
}

func tmp(){
    fmt.Println("y =", y)
}

The output: 

y = 12
y = 13
y = 14
y = 13
toe  = 0
foo  = 1
bar  = 2
apple  = 3
i = 5
green  = Four
Im  = false
Cookie  = Oatmeal
k = Not typed

The For Loop. The loop that does it all in Go

No more while loops, do while loops, etc, the 'for' loop does it all in go. Notice there are no parenthesis, and brackets are required even if you're looping over one line.

package main

import "fmt"

func main() {
    sum := 0
    for i:=0; i<100; i++ {
        sum+=i
    }   
fmt.Println("Sum: ", sum)
}

The output: 

Sum:  4950

Using For to make a While loop

What a simple idea, use one loop, syntax and format for everything. To get a while loop, just drop the first and last arguments off of your for loop and tada! Once again, no parenthesis, and brackets are required. 

package main

import "fmt"

func main() {
    sum := 0
    for sum < 100 {
        sum++
    }   
fmt.Println("Sum: ", sum)
}

The output: 

Sum:  100

If/Else Statements

Looking at the first if/else, you'll notice we're following the same format as before: no parenthesis, and brackets are required. Also, as you'll see in the second if/else block, if statements in go can include an initialization statement before the conditional, just like for loops. However, be aware that that changes the scope of that variable, again, just like for loops.

package main

import "fmt"

func main() {
    i := 10
 
    if i < 50 {
        fmt.Println("\"i\" is SMALLER than 50.")
        fmt.Println("Value of \"i\" inside 1st loop: ", i)
    } else {
        fmt.Println("\"i\" is BIGGER than 50.")
    }
    fmt.Println("Value of \"i\" after 1st loop: ", i)

    //more compact
    if i := i*6; i < 50 {
        fmt.Println("\"i\" is SMALLER than 50.")
    } else {
        fmt.Println("\"i\" is BIGGER than 50.")
        fmt.Println("Value of \"i\" inside 2nd loop: ", i)
    }
    fmt.Println("Value of \"i\" after 2nd loop: ", i)
}

The output: 

"i" is SMALLER than 50.
Value of "i" inside 1st loop:  10
Value of "i" after 1st loop:  10
"i" is BIGGER than 50.
Value of "i" inside 2nd loop:  60
Value of "i" after 2nd loop:  10

Time for a function call

Just like you use the keywords cons or var to create a variable, you use the keyword func to create a function. Also, if you haven't noticed already, declaring variables is backwords in go compared to other languages; meaning, first you write the name of the variable then you specify it's type. You'll see the same format below in function parameters and return values. You'll notice this function takes what go calls a slice (an array) as a parameter. A slice points to an array of values and has a size. We use the size in the for loop in the function. The last thing to point out is that you list the return type at the end of the function heading, right before the opening bracket, as you can see in the example.

package main

import "fmt"

func sum(seq []int) int {
    var value=0
    for i:=0; i<len(seq); i++ {
        value+=seq[i]
    }   
    return value
}

func main() {
    var seq=[]int{4,6,3,2,9,6}
    fmt.Println("Sum: ", sum(seq))
}

The output: 

Sum: 30

Import another package/Newton's method for finding square roots

This example demonstrates a few new things. Notice the import statement can be expanded with parenthesis. The local sqrt function also returns two items instead of one. Notice how the types for each value returned are declared right before the open curly bracket. Also, notice that calls to functions from a package begin with a capital letter. Last, we use printf for the first time here, which you've probably seen before, however, here we use a %v which stands for value and really simplifies things. 

package main

import (
    "fmt"
    "math"
)

func sqrt(x float64) (float64, int) {
    var curr, prev, n = 1.0, 0.0, 0

    for math.Abs(curr-prev) > .000000000000001 {
        prev = curr
        curr = curr-(math.Pow(curr, 2) - x)/(2*curr)
        n++ 
    }   
    return curr, n
}

func main() {
    var ans, n = sqrt(2)
    fmt.Println("Newton's Method: ", ans, "Iterations: ", n)
    fmt.Printf("Answer: %.16v\n", math.Sqrt(2))
}

The output: 

Newton's Method:  1.414213562373095 Iterations:  6
Answer: 1.414213562373095
