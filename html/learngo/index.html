Description and History:

Write quickly. Build quickly. Run quickly.

Go is a compiled, garbage collected, concurrent language developed by Google. Specifically, the language was developed in 2007 by Robert Griesemer, Rob Pike (whose name you may recognize from the book, "The Unix Programming Environment"), and Ken Thompson (who worked with Dennis Ritchie to create Unix). If you're coming from the likes of C or Java, you'll recognize that much of the syntax and the style, while different, will not seem very foreign to you. However, one of Go's strengths is that it takes out a lot of the messy stuff in C, and creates a more consistent, easier to learn, and surprisingly elegant language. For example, Go has pointers, but no pointer arithmetic. Along the same vein of simplifying things, the ++ and -- operators are statements in go and not expressions. You'll find that out if you try and increment a variable inside of a Println function. Go also only allows the postfix use of those operators on top of not being able to embed them into expressions. This simplifies the syntax and gets rid of the messy issues surrounding order of operation. Further, you may be surprised initially that Go doesn't have the keywords: do, do while, or while, to make loops. Instead, Go uses a beefed up for loop that accomplishes all those things in one. As you dive into Go and explore it's design and use, I think you may agree that the one theme that persists is simply the idea or end goal that software should be simple.

Installing Go:

Give it a try. It's pretty easy to get Go-ing.

  • Click here to install go.
  • To make sure it's working properly, we'll make a short test program. Create a new file named hello.go and type the following:

    • package main

import "fmt"

func main() {
    fmt.Printf("Hello World!\n")
}
  • Run your program as follows:

    • $ go run hello.go
Hello World!
  • You can also build your program and it will create an executable file with the same name as the source code file that was built:

    • $ go build hello.go
$ ./hello
Hello World!

    If it prints the message "Hello World!" then you'll know everything is working. Congratulations, you just built your first Go program!

    Setting Up Vim

    Since Go is a new language, next you'll probably want to set up Vim with the go plugins and syntax highlighting. The following instructions are from the readme.txt file you'll be able to find in the ../misc/vim/ directory where go was installed. Here are the instructions for editing your ".vimrc" file:

    #To use all the Vim plugins, add these lines to your $HOME/.vimrc.

# Some Linux distributions set filetype in /etc/vimrc.
# Clear filetype flags before changing runtimepath to force Vim to reload them.
if exists("g:did_load_filetypes")
  filetype off
  filetype plugin indent off
endif
set runtimepath+=$GOROOT/misc/vim    # replace $GOROOT with the output of: go env GOROOT
filetype plugin indent on
syntax on

    Now, open hello.go back up in vim and see how pretty it is.

    Using Gccgo

    For some reason, getting things going with gccgo was quite a bit harder than I anticipated it would be. I read golang's page here, specifically, the section here, but I didn't have much luck at getting any programs compiled to the point where I could run them. Using the -c flag, I could get the comiple object code, and I used the -o flag to link the object code and create an executable, but every time I tried to run the executable, I got this error:

    [msaunder@onyx go]$ gccgo -c hello.go 
[msaunder@onyx go]$ gccgo -o hello hello.o
[msaunder@onyx go]$ ./hello 
no debug info in ELF executable errno -1
fatal error: no debug info in ELF executable

    Finally, I was able to get the answer from professor Jim Buffenbarger at Boise State University, who lead us to the -g flag that worked to succesfully compile our go programs. The first way produces the classic a.out executable file, and the second method uses the -o flag where you can specify a name for your output file: 

    [msaunder@onyx go]$ gccgo -g hello.go 
[msaunder@onyx go]$ ./a.out 
Hello World!
[msaunder@onyx go]$ gccgo -g -o hello hello.go
[msaunder@onyx go]$ ./hello 
Hello World!

    Mergesort Code. This should help sort things out.

    Mergesort is an O(n*logn) comparison-based sorting algorithm you've probably already seen or at least heard of before. It works by repeatedly breaking down an unsorted list into single 1 element sorted items. In that state, it compares the "sorted" sublists and merges everything back together resulting in a final sorted list.

    Linked List Code. C'mon, Go-pher It.

    Hyperbolically speaking, the Double-Linked List is probably every new Computer Science students most favorite data structure! A Linked List is essentially a list composed of individual nodes that generally hold an item of data and also information about how to get to the next or previous node in the list.
